Case Study

아픈 반려견의 하루를
기록하는 펫케어 웹앱

노령견이나 질환이 있는 반려견은 약, 식사, 물, 증상, 진료 일정처럼 챙길 게 많습니다. PetCare+는 이 기록들을 한 화면에 모아, 매일 반복하는 케어를 빠뜨리지 않고 병원 가기 전에 한눈에 정리할 수 있게 만든 웹앱입니다.

Background

돌봄은 감정의 영역이지만, 동시에 매일의 기록 관리이기도 합니다.

아픈 반려견을 돌보는 보호자는 약 먹은 시간, 증상 변화, 병원 일정, 검사 결과를 계속 챙겨야 합니다. 그런데 이 기록들은 보통 메신저, 메모장, 사진첩, 병원 안내지에 흩어져 있어 뭔가 하나는 꼭 빠지게 됩니다.

왜 만들었나

체크리스트로 끝내는 게 아니라, 매일 남긴 기록이 병원 상담과 건강 추적에 그대로 쓰이면 좋겠다고 생각했습니다. 특히 노령견 보호자는 작은 변화에도 불안해지기 때문에, 경고를 앞세우기보다 차분하게 확인하고 넘어가는 흐름에 집중했습니다.

01
기록이 흩어져 있다 약, 증상, 사진, 병원 일정이 각기 다른 앱에 저장돼 전체 흐름을 한 번에 보기 어렵습니다.
02
매일 하는 일이라 빠뜨리기 쉽다 약, 식사, 물 챙기기는 단순하지만 한 번 놓치면 건강에 바로 영향을 줍니다.
03
진료 전마다 다시 정리해야 한다 병원 가기 전 증상 변화와 검사 결과를 매번 모으느라 보호자의 부담이 큽니다.
생활 사진 약, 물그릇, 진료 수첩을 함께 챙기는 보호자의 일상 장면

Goal

오늘 할 일을 놓치지 않고, 기록은 부담 없이 남기게.

기능을 많이 넣기보다 케어 루틴의 핵심 동작을 빠르게 확인하고 기록하는 흐름을 만드는 데 집중했습니다. MVP에서는 반려견 한 마리 기준으로 홈, 일정, 기록, 리포트 구조를 먼저 잡았습니다.

01

오늘의 케어를 먼저

약, 식사, 물, 증상, 체중처럼 매일 챙길 항목을 홈에서 바로 체크합니다.

02

진료 전 한눈에

병원 가기 전 최근 증상과 검사 결과를 빠르게 확인하도록 기록 중심으로 구성했습니다.

03

무겁지 않게

질환 관리 앱 특유의 무거움을 덜기 위해 차분한 색, 큰 카드, 짧은 문장을 썼습니다.

04

키워갈 수 있게

지금은 한 마리에 집중하고, 이후 다견 관리·보호자 공유·AI 요약으로 넓힐 수 있게 설계했습니다.

Insight

문제는 기록의 양이 아니라, 필요할 때 다시 찾기 어려운 구조였습니다.

기능을 늘리는 대신 보호자가 실제로 반복하는 동작을 기준으로 화면을 다시 짰습니다. 입력이 쉬워야 기록이 이어지고, 이어진 기록이 결국 진료와 회고에 쓰입니다.

문제
케어 항목이 흩어져 있다

약, 식사, 증상, 병원 일정이 따로 놀아서 매번 기억에 의존해야 했습니다.

감정적 피로가 쌓인다

아픈 아이를 돌볼 땐 작은 누락도 불안으로 번지기 때문에 화면의 압박을 덜어야 했습니다.

해결
홈에서 바로 체크

오늘 할 케어를 홈에서 먼저 보여주고, 끝난 항목은 흐름에서 조용히 정리되게 했습니다.

기록과 리포트를 잇기

입력으로 끝나지 않고 날짜별 기록과 건강 요약으로 이어지도록 구조를 연결했습니다.

배운 점
오래 머무는 앱이 아니다

케어 앱은 필요할 때 빠르게 열어 기록하고 닫는 쪽이 더 맞다고 봤습니다.

의료감보다 생활감

전문성을 앞세우기보다 일상에 자연스럽게 스며드는 톤이 필요했습니다.

Design System

차분한 톤은 지키되, 실제로 만들 수 있는 구조로 정리했습니다.

불안을 자극하지 않는 색과 여유 있는 카드 레이아웃을 기준으로 시스템을 잡았습니다. 개발은 HTML/CSS/JS로 시작해, 이후 Supabase로 데이터를 연결할 수 있게 흐름을 미리 고려했습니다.

비주얼

메인은 세이지 그린 계열로 안정감을 줬습니다. 카드 UI로 정보 묶음을 구분하고, 모바일에서 빠르게 훑을 수 있게 여백과 대비를 넉넉히 잡았습니다.

UI 컴포넌트 케어 카드, 일정 카드, 기록 입력 카드, 상태 태그, 진행률 바를 재사용 가능한 단위로 만들었습니다.
바이브 코딩 Codex와 Claude로 화면 구조와 Supabase 연동, 입력·조회 로직을 단계별로 구현했습니다.

디자인 시스템 / 목업 Figma 컬러·컴포넌트 정리와 주요 모바일 화면 3~5개 목업

Result

MVP 기준의 핵심 화면과 데이터 흐름을 결과물로 정리했습니다.

단순 시안이 아니라 구현과 확장을 염두에 둔 웹앱 구조로 만들었습니다. 문제 정의부터 화면 설계, 디자인 시스템, 개발까지 하나의 흐름으로 이어지게 구성했습니다.

RESULT 01 핵심 IA 정리

프로필, 오늘의 케어, 일정, 기록, 리포트 구조를 MVP 기준으로 정리했습니다.

RESULT 02 모바일 웹앱 UI

자주 보는 정보를 카드 중심으로 배치하고 모바일 사용성을 우선했습니다.

RESULT 03 개발 연결

정적 화면에서 시작해 저장·조회·완료 처리 같은 실제 서비스 흐름까지 다뤘습니다.

최종 화면 목업 홈·기록·일정·리포트 화면을 4분할로 배치한 결과물

Review

좋은 케어 서비스는 기능 수가 아니라, 매일 반복할 수 있는 구조에서 시작된다는 걸 배웠습니다.

UI 디자인이 화면을 예쁘게 만드는 데서 끝나지 않고, 사용자의 감정과 반복 행동까지 함께 설계하는 일이라는 걸 다시 느꼈습니다. 특히 반려동물 건강처럼 예민한 주제에서는 경고와 정보의 강도를 조절하는 게 중요했습니다. AI 코딩 도구를 쓰면서도, 기획자가 화면 구조와 데이터 흐름을 명확히 잡아야 결과가 안정적으로 나온다는 점을 확실히 알게 됐습니다.
